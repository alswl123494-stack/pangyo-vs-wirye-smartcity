## 데이터 출처 및 처리 과정

| 데이터 | 출처 | 기준시점 |
|---|---|---|
| 지하철 네트워크 그래프 | 강의 제공 `subway_network.zip` (역/구간/환승, README 포함) | 2026-05-04 ("현재" 스냅샷) |
| 집계구 인구(총인구) | SGIS 통계지리정보서비스 — 인구총괄 | 2023년 |
| 집계구 종사자·사업체수 | SGIS 통계지리정보서비스 — 산업분류별(10차 대분류) 총괄 | 2023년 |
| 집계구 경계 | SGIS 통계지리정보서비스 — 집계구경계(센서스용) | 2025년 2분기(6월 30일 기준) |

### 처리 단계 (저장소의 분석 스크립트, 실행 순서)

1. `isochrone_analysis.py` — 지하철 네트워크 그래프에서 다익스트라 최단시간 탐색 → 판교역·청라국제도시역 기준 30분/60분 등시간권 폴리곤(`isochrone_*.geojson`) 및 역 단위 소요시간 테이블(`reachability_times.csv`) 생성
2. `build_curve_population.py` — SGIS 집계구 경계 + 인구·종사자 통계를 결합해, 각 집계구를 700m 이내 가장 가까운 역에 배정하고 그 역의 소요시간을 집계구의 도달시간으로 사용 → 0~60분 누적 인구/종사자 곡선(`curve.json`)과 30·60분 스냅샷 통계표(`stats.json`) 생성

### 핵심 가정과 한계

- 제공된 네트워크 데이터는 지하철만 포함하고 보행망이 없음 → 등시간권 폴리곤은 "도달 가능한 역의 700m 도보 버퍼 합집합"으로 근사함. 실제 도보 네트워크 기반 등시간권보다 거친 근사치임.
- 집계구는 가장 가까운 역(700m 이내)에 단순 배정하는 방식이라, 두 역의 영향권이 겹치는 경계 지역에서는 약간의 오차가 있을 수 있음.
- 인구/종사자 기준연도(2023)와 네트워크 기준시점(2026-05)이 다름 — 망 구조 자체는 거의 변화가 없다고 가정.

## AI 활용 내역

데이터 전처리 스크립트 작성, 다익스트라 분석 코드, 웹앱(Leaflet/Chart.js) 프론트엔드 코드를 Claude(Anthropic)와 함께 작성함. 산출된 수치는 원본 데이터로 재검증함.
